## 💡 什么是 Folio?

Folio 是一款旨在极大提升媒体笔记记录效率的 Obsidian 插件。它的核心价值在于**消除了从“发现作品”到“实际动笔记录”之间的所有摩擦**。

通过将来自全球数据源（如豆瓣、IMDB）的丰富结构化数据与你**自定义的模板**相结合，Folio 能瞬间为你的评论或影评构建出一个专业的基础框架。你将同时获得两者的优势：一方面是标准、详尽的外部数据，另一方面是你个性化的笔记布局。笔记添加完成后，你唯一的任务就是集中精力思考并写下你独特的见解。

---

## ✨ 核心特性

- 📑 **数据驱动模板**：把无聊的琐事交给插件。通过定义你自己的笔记模板，你可以设置占位符字段（例如 `{{title}}`、`{{author}}`、`{{cover_url}}`）。Folio 会从外部数据源抓取准确的数据并注入到模板中，确保每一篇新笔记都立即呈现你喜欢的样式。
- 🌏 **智能多源路由**：无需手动指定搜索位置。它会自动检测查询中的中日韩（CJK）或拉丁字符，从而智能地将请求发送至最相关的数据库——中文媒体使用豆瓣，英文/西方媒体使用 IMDB 或 Open Library。
- ⚡ **无缝工作流**：一个命令 (`Folio: Add Note`) 搞定一切。发现一本书，添加到 Obsidian，几秒钟内笔记就准备好开写。它完全省去了构建结构化媒体日志时繁琐的手动复制粘贴过程。
- 🛠️ **高度自定义系统**：让插件符合你的笔记习惯。自由选择存放新笔记的 Inbox 文件夹，设置请求延迟以尊重频率限制，并为不同的媒体类型设计多个模板。
- 💾 **本地优先缓存**：始终保持高性能。Folio 内置了本地缓存机制。一旦书籍或电影信息被抓取，其元数据将存储在你库（Vault）的本地空间中，避免重复的 API 调用，确保你的笔记创建流程始终流畅。

## 📦 安装方法

> [!IMPORTANT]
> 本插件目前仅支持 **桌面端 (Desktop-only)**。

1. **从源码构建**:
   ```bash
   npm install
   npm run build
   ```
2. **复制文件**:
   在你的库中创建文件夹：`<vault>/.obsidian/plugins/obsidian-folio/`。
   将 `main.js`、`manifest.json` 和 `styles.css` 复制到该文件夹中。
3. **启用**:
   进入 **设置 (Settings) → 社区插件 (Community Plugins)**，点击 **重载 (Reload)**，找到 **Folio** 并开启。

---

## ⚙️ 配置选项

前往 **设置 (Settings) → Folio** 配置以下内容：

| 设置项 | 描述 |
| :--- | :--- |
| **Firecrawl API key** | *可选*。抓取豆瓣详细数据时需要。 [获取 API Key](https://www.firecrawl.dev/) |
| **Inbox folder** | 新笔记创建的目标子文件夹（例如：`inbox`）。 |
| **Request delay** | 请求间隔秒数（默认：2），用于避免触发数据源的频率限制。 |

## 🚀 使用指南

1. 打开 **命令面板** (`Ctrl/Cmd + P`)。
2. 运行命令：`Folio: Add Note`。
3. 在弹窗中输入查询内容：
   * **Search**: 书名、电影名、作者或关键词。
   * **ISBN**: 用于图书的精确查找（跳过标题搜索）。
4. **数据源逻辑**:
   * **中日韩字符 (CJK)**: 搜索 豆瓣 + Google Books。
   * **拉丁字符 (Latin)**: 搜索 IMDB + Open Library。
   * **ISBN**: 在所有兼容数据源中并行搜索。

## ⚠️ 网络声明
根据 Obsidian 开发者政策，本插件会连接以下外部服务：
* **服务项目**: Douban, IMDB, Open Library, Google Books, 以及可选的 Firecrawl。
* **隐私声明**: 仅搜索查询内容和 ID 会发送至上述服务。插件绝不会上传你的本地库数据或个人笔记。

---

## 🛠️ 常见问题排查

* **豆瓣字段为空**: 请检查设置中的 Firecrawl API key 是否有效且填写正确。
* **文件夹错误**: 请确保设置中定义的 "Inbox folder" 在你的库中确实存在。
* **数据过时**: 删除 `.obsidian/plugins/obsidian-folio/cache.json` 中的相关条目即可强制重新抓取。
